## STEP 2: Deploy Backend to Render

1. Go to https://render.com
2. Click "New +" → "Web Service"
3. Select your GitHub repo: DINAL11/3D-Word-Cloud-Dinal
4. Configure:
   - **Name**: 3D-Word-Cloud-Backend
   - **Root Directory**: Backend
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r requirements.txt && python -c "import nltk; nltk.download('punkt'); nltk.download('stopwords'); nltk.download('punkt_tab')"`
   - **Start Command**: `uvicorn main:app --host 0.0.0.0 --port $PORT`
5. Add Environment Variable:
   - **Key**: FRONTEND_URL
   - **Value**: https://your-vercel-frontend-url.vercel.app (update after Vercel deployment)
6. Click "Create Web Service"
7. Wait for deployment to complete
8. **COPY the Render URL** (e.g., https://3d-word-cloud-backend.onrender.com)

## STEP 3: Deploy Frontend to Vercel

1. Go to https://vercel.com
2. Click "Add New" → "Project"
3. Import your GitHub repo: DINAL11/3D-Word-Cloud-Dinal
4. Configure:
   - **Framework Preset**: Other (detect automatically)
   - **Root Directory**: frontend
   - **Build Command**: `npm run build`
   - **Output Directory**: dist
   - **Install Command**: `npm install`
5. Add Environment Variables:
   - **Key**: VITE_API_URL
   - **Value**: Paste the Render backend URL from STEP 2 (e.g., https://3d-word-cloud-backend.onrender.com)
6. Click "Deploy"
7. Wait for deployment to complete
8. **COPY the Vercel URL** (e.g., https://3d-word-cloud-dinal.vercel.app)

## TROUBLESHOOTING

**Q: Frontend shows "Failed to analyze article"**
A: Check that VITE_API_URL environment variable is set to your Render backend URL in Vercel

**Q: Getting CORS errors**
A: Make sure Backend's FRONTEND_URL environment variable matches your Vercel URL exactly

**Q: Port already in use locally**
A: Run `lsof -ti:8000 | xargs kill -9` to free port 8000
